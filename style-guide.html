09. Accessibility & Responsive Design

Keyboard Navigation

  • All interactive elements are keyboard accessible
  • Focus visible: 3px burgundy outline with 2px offset
  • Skip to main content link (hidden until focused)
  • Logical tab order throughout

Motion Preferences

  • Respects prefers-reduced-motion
  • Animations disabled for motion-sensitive users
  • Transitions reduced to 0.01ms when requested
  • Profile morphing animation paused

Semantic HTML

  • <address> for contact information
  • <article> and <time> for experience
  • <ul> with role="list" for skills
  • Proper heading hierarchy (h1, h2, h3)

ARIA Labels

  • Contact links include descriptive labels
  • External links marked appropriately
  • Screen reader friendly navigation
  • Location marked with proper context

10. Touch Device Optimization

Media Query: @media (hover: none) and (pointer: coarse)

On touch devices, transform effects are disabled to prevent awkward interactions:

  • Contact buttons: No scale transform on hover
  • Skill bubbles: No scale transform on hover
  • Bento items: No translateY on hover
  • Experience cards: No transform effects

This ensures a smooth, native-feeling experience on mobile devices while maintaining visual feedback through color changes.
